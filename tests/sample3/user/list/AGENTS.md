# ユーザーマスタ検索画面 設計書

## 1. 画面概要

ユーザーマスタ (`user_master`) を検索し、一覧表示する画面。
ユーザーコード、ユーザー名、メールアドレス、所属組織、利用状態を条件に検索する。

## 2. 対象テーブル

- `ddl/user_master.sql`

## 3. 検索条件

| 項目名 | 物理名 | 型 / 桁数 | 入力形式 | 説明 |
| --- | --- | --- | --- | --- |
| ユーザーコード | `user_code` | `VARCHAR(20)` | テキストボックス | 前方一致または部分一致検索 |
| ユーザー名 | `user_name` | `VARCHAR(100)` | テキストボックス | 部分一致検索 |
| メールアドレス | `email` | `VARCHAR(255)` | テキストボックス | 部分一致検索 |
| 組織ID | `organization_id` | `BIGINT` | プルダウン / 組織検索ダイアログ | 所属組織で絞り込み |
| 有効フラグ | `is_active` | `BOOLEAN` | ラジオボタン / セレクト | `全て / 有効 / 無効` |

## 4. 一覧表示項目

| 項目名 | 物理名 | 説明 |
| --- | --- | --- |
| ユーザーID | `user_id` | DB採番ID |
| ユーザーコード | `user_code` | 一意キー |
| ユーザー名 | `user_name` | 表示名 |
| メールアドレス | `email` | 連絡先 |
| 組織ID | `organization_id` | 所属組織 |
| 有効フラグ | `is_active` | 利用状態 |
| 作成日時 | `created_at` | 登録日時 |
| 更新日時 | `updated_at` | 更新日時 |

## 5. 操作

- `検索` ボタン
  入力条件に一致するユーザーを一覧表示する
- `クリア` ボタン
  検索条件を初期化する
- `新規登録` ボタン
  ユーザーマスタ登録画面へ遷移する
- 一覧行クリック
  将来的に詳細 / 編集画面へ遷移可能とする

## 6. 検索仕様

- `user_code` は前方一致または部分一致で検索する
- `user_name` は部分一致で検索する
- `email` は部分一致で検索する
- `organization_id` 指定時は完全一致で検索する
- `is_active` が `全て` の場合は条件に含めない
- 初期表示は未検索、または有効データのみ表示のいずれかを採用する

## 7. ソート順

- 初期ソートは `user_code ASC`
- 任意で `updated_at DESC` への切替を許可する

## 8. メッセージ

- 検索結果なし: `該当するユーザーはありません。`
- 検索失敗: `検索処理でエラーが発生しました。`

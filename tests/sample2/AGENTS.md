# 商品マスタ検索画面 UI設計書

共通方針は [../AGENTS.md](/home/hidek/git/design-system-skills/tests/AGENTS.md) を参照。

## 1. 目的

商品マスタ検索画面は、商品コード、JANコード、商品名、カテゴリ、販売状態などの条件から目的の商品を素早く特定し、詳細確認や編集画面への導線を提供することを目的とする。

主な利用者は以下を想定する。

- 商品管理担当者: 商品情報の確認、修正対象の特定を行う
- 受発注・在庫担当者: 販売状態、在庫管理有無、単価などを確認する
- 管理者: 商品マスタ全体の整合性や登録状況を把握する

## 2. 画面ゴール

- 商品を複数条件で絞り込み、目的のレコードへ短時間で到達できる
- 商品の販売状態、カテゴリ、単価、在庫管理有無などの主要情報を一覧で比較できる
- 検索結果から商品詳細または編集画面へ1クリックで遷移できる
- 条件の再利用や絞り込みのやり直しがしやすい

## 3. 画面構成

商品マスタ検索画面は以下の6領域で構成する。

1. グローバルヘッダー
2. サイドナビゲーション
3. 画面ヘッダー / 条件サマリー
4. 検索条件エリア
5. 検索結果一覧エリア

補足要件:

- 上記6領域はHTMLモックにおける必須構成要素とする
- デザインシステム適用時も、情報設計としてヘッダーおよびメニュー導線は省略しない
- 固定サイドナビが不自然な場合でも、主要メニュー導線はグローバルヘッダーまたは代替ナビゲーションとして常に視認可能にする
- 画面中央の検索フォームと結果一覧だけで完結する構成は不可とし、画面タイトル、メニュー導線、補助情報領域を持たせる

## 4. レイアウト

### PC

- 画面上部に固定ヘッダー
- 左側に固定サイドナビ
- メインエリア上段に画面タイトル、検索件数、主要操作を配置
- 中段に検索条件フォーム
- 下段に検索結果テーブル
- 右下または下段末尾に検索履歴、保存条件、注意事項などの補助情報を配置

### タブレット

- サイドナビは折りたたみ式
- 検索条件は2列またはアコーディオン表示
- テーブルは主要列優先で横スクロール許容
- 補助情報は検索結果の下へ縦積み

### モバイル

- ヘッダー + ハンバーガーメニュー
- 検索条件は縦並び
- 検索結果はカード表示または簡略テーブル
- 条件変更、検索、リセットを画面上部で実行可能にする

## 5. UIコンポーネント設計

### 5.1 グローバルヘッダー

表示要素:

- システム名
- 画面タイトル
- 全体検索
- 通知アイコン
- ユーザー情報

表示サンプル:

- システム名: `マスタ管理システム`
- 画面タイトル: `商品マスタ検索`
- 検索プレースホルダ: `商品コード・JANコード・商品名で検索`
- 通知: `未読 2 件`
- ユーザー名: `山田 花子`
- 所属/権限: `商品管理部 / 管理者`
- 主要操作例: `新規商品登録`, `CSV出力`, `検索条件保存`

要件:

- 通知未読件数をバッジ表示
- 全体検索は商品コード、JANコード、商品名の簡易検索導線としても使える
- ユーザーメニューからプロフィール、権限情報、ログアウトへ遷移可能
- デザインシステム差異にかかわらず、グローバルヘッダー自体は必ず表示する

### 5.2 サイドナビゲーション

メニュー例:

- ダッシュボード
- 商品マスタ検索
- 商品カテゴリ管理
- 承認待ち
- 更新履歴
- 監査ログ
- 設定

表示サンプル:

- `ダッシュボード`
- `商品マスタ検索`
- `商品カテゴリ管理`
- `承認待ち`
- `更新履歴`
- `監査ログ`
- `設定`

補助表示サンプル:

- 商品マスタ検索: `active`
- 承認待ち: `3`
- 更新履歴: `new`

要件:

- 現在地を明確にハイライト
- PCでは原則として永続表示のサイドナビ、またはそれと同等の主要メニュー領域を持つ
- 商品マスタ検索、カテゴリ管理、履歴系への導線は必ず残す

### 5.3 画面ヘッダー / 条件サマリー

表示内容:

- 画面タイトル
- 検索結果件数
- 現在の条件サマリー
- 主要操作

表示サンプル:

- タイトル: `商品マスタ検索`
- 件数: `248 件`
- 条件サマリー: `ACTIVE / 食品 / 在庫管理あり`
- 操作: `検索`, `条件クリア`, `検索条件保存`

要件:

- 検索実行後は件数を強調表示
- 現在の主な絞り込み条件をチップまたはテキストで表示
- 条件クリアは1アクションで実行できる

### 5.4 検索条件エリア

検索項目:

- 商品コード
- JANコード
- 商品名
- 商品名カナ
- 商品略称
- カテゴリ
- 商品種別
- 単位
- 標準単価 下限 / 上限
- 原価 下限 / 上限
- 税区分
- 税率
- 在庫管理フラグ
- シリアル管理フラグ
- ロット管理フラグ
- 賞味期限日数 下限 / 上限
- 発売日 From / To
- 販売終了日 From / To
- ステータス
- 更新者
- 更新日時 From / To

入力形式:

- コード系はテキスト入力
- 名称系は部分一致を想定したテキスト入力
- カテゴリ、商品種別、単位、税区分、ステータスはセレクト
- 管理フラグはチェックボックスまたはトグル
- 金額、税率、賞味期限日数は数値範囲入力
- 日付、日時は期間指定

要件:

- 主要条件を初期表示し、詳細条件は折りたたみ可能にしてよい
- `product_type` は `NORMAL`, `SERVICE`, `SET`, `OPTION`
- `tax_type` は `TAXABLE`, `REDUCED`, `EXEMPT`, `NON_TAXABLE`
- `status` は `ACTIVE`, `SUSPENDED`, `ENDED`
- 発売日と販売終了日の逆転条件は入力時に警告表示する
- 数値範囲は下限 > 上限 の場合にエラー表示する

### 5.5 検索結果一覧エリア

推奨カラム:

- 商品コード
- JANコード
- 商品名
- 商品略称
- カテゴリ名
- 商品種別
- 単位
- 標準単価
- 原価
- 税区分
- 税率
- 在庫管理
- ステータス
- 発売日
- 販売終了日
- 最終更新日時
- 更新者
- 操作

操作:

- 詳細
- 編集
- 履歴

要件:

- ソート、フィルタ、ページネーション対応
- ステータスはタグで視認しやすく表示
- `ACTIVE`, `SUSPENDED`, `ENDED` は色とラベルで区別
- 在庫管理、シリアル管理、ロット管理はアイコンまたは `あり/なし` で表示
- 行クリックでも詳細に遷移可能
- 重要列として商品コード、商品名、ステータスは先頭側に固定

## 6. 画面内の主要操作フロー

### フロー1: 商品コードで対象商品を検索する

1. 商品コードを入力
2. 検索を実行
3. 検索結果一覧から対象商品を確認
4. 詳細または編集を選択

### フロー2: 複数条件で絞り込む

1. カテゴリ、ステータス、在庫管理有無を指定
2. 必要に応じて価格帯や発売日範囲を追加
3. 検索を実行
4. 件数と条件サマリーを確認
5. 条件を微調整して再検索

## 7. 状態設計

### 初期状態

- 主要検索条件のみ表示
- 結果一覧は未検索状態メッセージを表示

### ローディング

- 検索実行中は条件エリアの検索ボタンをローディング表示
- 結果一覧はスケルトンまたはローディングインジケータを表示

### 検索結果なし

- `条件に一致する商品はありません` を表示
- 条件を緩める提案や `条件クリア` 導線を表示

### 入力エラー

- 不正な範囲指定や日付逆転は項目単位でエラーメッセージを表示
- エラーがある場合は検索実行を抑止する

## 8. 権限設計

- 閲覧者: 検索と詳細確認のみ可能
- 編集者: 詳細確認と編集が可能
- 管理者: 新規登録、編集、CSV出力、検索条件保存が可能

要件:

- 権限に応じて `編集`, `新規商品登録`, `CSV出力` の表示可否を制御する
- 検索対象そのものは権限に応じてカテゴリやステータスで制限される場合がある

## 9. アクセシビリティ / UX要件

- ラベルは省略せず、プレースホルダだけに依存しない
- キーボード操作で検索条件入力、検索実行、一覧操作が可能
- ステータスやフラグは色だけでなくラベルやアイコンでも区別する
- テーブル横スクロール時も主要列の意味が失われないようにする
- モバイルでは検索条件の開閉状態を明確にする
